# KMSWC — Kinetic Model of Solid Waste Composting

A Python desktop application that simulates a **two-stage, six-layer composting process** based on the kinetic model of Bari & Koenig (2012). It computes and visualises temperature, water content, volatile solids, and oxygen concentration hour-by-hour over two sequential 28-day composting stages.

This software was developed as part of the research paper:

> **Development of a Programming Based Kinetic Model for Two Stage Composting of Solid Waste**  
> Zarif Tanzim Aziz, Md. Mahfuzur Rahman, Quazi Hamidul Bari  
> Khulna University of Engineering & Technology / Ahsanullah University of Science and Technology  
> arXiv:2606.24556 — <https://arxiv.org/abs/2606.24556>

---

## What it does

The composting pile is modelled as a stack of six layers through which air flows continuously. At each hourly time step the model tracks, for every layer:

- **Temperature** — heat generated by biodegradation vs. heat lost to airflow and surroundings
- **Volatile Solids (VS%)** — biological degradation rate driven by temperature and oxygen
- **Water Content (WC%)** — moisture gained from vapour in inflowing air, lost with outflowing air
- **Oxygen (O₂%)** — O₂ consumed by microbial activity

**Stage 1** runs for 28 days with the supplied air-flow schedule. At the end of Stage 1 the compost has dried significantly. **Stage 2** re-wets the pile to 60 % moisture and runs a further 28 days to simulate continued degradation. Stage 2 inherits the final solid fractions from Stage 1 automatically.

Outputs include an initial/final mass summary, layer heights before and after, and six graphs.

## Requirements

- Python 3.9 or newer
- `numpy`
- `matplotlib`
- `tkinter` (ships with Python on Windows; install `python3-tk` on Linux)

Install the two third-party libraries with:

```bash
pip install numpy matplotlib
```

## Running the program

Run the following command from the **project root** (the folder that *contains* the `kmswc` package folder):

```bash
python -m kmswc
```

The GUI window opens. Workflow:

1. Enter the **28 daily air-flow rates** (m³ m⁻² h⁻¹) in the left column.
2. Fill in the **14 input parameters** in the middle column (temperatures, coefficients, initial mass fractions, etc.).
3. Click **RUN** to compute Stage 1. Results and six graphs appear immediately.
4. Scroll to the **Stage 2** section, adjust parameters if needed, and click its **RUN** button. Stage 1 must be completed first.

### Headless run (no window)

To run the model and print output tables in the terminal without opening the GUI:

```bash
python -m kmswc.examples.run_headless
```

Edit the input values near the top of [examples/run_headless.py](examples/run_headless.py) to use different parameters.

## Input parameters

| Parameter | Symbol | Unit |
| --- | --- | --- |
| Initial temperature of composting mass | Tci | °C |
| Ambient temperature | Ta | °C |
| Heat energy from BVS degradation | HL | kJ/kg |
| Specific heat capacity of wet composting material | Cpc | J/kg·K |
| Specific heat transfer coefficient | kc | W/m²·K |
| Reaction rate | Khr | — |
| Relative humidity | RH | % |
| Initial mass of composting waste | I\_mass | kg/m² |
| Initial water content | PWC | % |
| Initial fixed solid | PFS | % |
| Initial non-biodegradable volatile solid | PNVS | % |
| Initial volatile solid | PVS | % |
| Density of composting solid waste | ρ | kg/m³ |
| Height of composting pile | h | cm |

## Project structure

```text
kmswc/
├── domain/              # Physics and mathematics (no GUI dependency)
│   ├── parameters.py    # Input dataclass and physical constants
│   ├── physics.py       # Hour-by-hour mass/energy/oxygen step (single implementation)
│   ├── series.py        # Time-series containers
│   ├── simulation.py    # Runs one layer; chains all six layers
│   └── mass_balance.py  # Final-mass and pile-height calculations
│
├── application/
│   └── stage_runner.py  # Orchestrates Stage 1 → Stage 2 handoff
│
├── presentation/        # GUI and visualisation (Tkinter + Matplotlib)
│   ├── gui.py           # Main window, input/output fields
│   └── plots.py         # Six-panel figure
│
├── examples/
│   └── run_headless.py  # CLI entry point for scripted use
│
└── __main__.py          # Entry point for `python -m kmswc`
```

The physics step is written **once** in `domain/physics.py` and called for all six layers across both stages — making it straightforward to inspect or modify any equation in a single place.

## Sample results

With the reference parameters from the paper (Tci = 25 °C, Ta = 25 °C, air flow = 5 m³ m⁻² h⁻¹, initial mass = 750 kg/m²):

| Parameter | Initial | After Stage 1 | After Stage 2 |
| --- | --- | --- | --- |
| Total composting mass (kg/m²) | 750.00 | 358.15 | 332.90 |
| Water mass (kg/m²) | 441.75 | 117.30 | 113.22 |
| Volatile solid mass (kg/m²) | 147.96 | 80.56 | 59.39 |

Overall VS reduction: ~46 % across both stages (~10 % in Stage 1, ~2 % in Stage 2).
